## AI 대화 기록 & 관찰 내용

**Q1: 이 코드에서 AABB를 화면에 그리는 부분만 설명해줘**

- Ai 답변: pygame.draw.rect(screen, RED, player, 2)
pygame.draw.rect(screen, RED, fixed, 2)

- 결과: 사각 도형에 딱 알맞는 AABB가 만들어져서 눈으로 확인 가능헀다.

**Q2: pygame.draw.rect 의 인자가 각각 뭘 의미하는지 포함해줘**

- Ai 답변: pygame.draw.rect(화면, 색상, 사각형, 두께) / screen → 그릴 대상 화면 / RED → 빨간색 (AABB 표시용) / player / fixed → 각각의 사각형 객체 / 2 → 선 두께 (0이면 채워짐, 2는 테두리만 그림)

- 배운 점:  pygame.draw.rect의 구조와 사용방법을 알게 되었다.

**Q3: 원형 충돌 감지에서 math.sqrt 를 쓰는 이유가 뭐야?**

- Ai 답변: math.sqrt → 두 점 사이의 실제 거리 계산
거리 ≤ 반지름 합 → 원형 충돌 판정
최적화: 제곱 비교로 sqrt 생략 가능

- 배운 점: 두 원의 중심점 간 x, y 거리 차이, 두 점 사이의 실제 직선 거리를 구하기 위해 피타고라스 정리 사용하여 두 원 중심 사이의 실제 거리를 계산을 하기위해 쓰는 것은 알았다.

**Q4: sqrt 없이 구현할 수 있어? 차이가 뭐야?**

- Ai 답변: sqrt 없이 구현하면 연산 속도가 조금 더 빨라집니다.
차이점
sqrt 사용 : 계산-math.sqrt(dx² + dy²) / 비교-distance <= r1+r2 / 속도-조금 느림
sqrt 없이 : 계산-dx² + dy²만 계산 / 비교-dx²+dy² <= (r1+r2)² / 속도-조금 빠름 (프레임 수 많을 때 체감)

- 배운 점: math.sqrt()는 함수 호출 + 제곱근 계산이 필요해서, 게임에서 매 프레임 수천 번 호출하면 약간 부담이다. 오브젝트가 많거나 실시간 물리 계산이 많으면 sqrt 없이 구현하는 게 효율적이란 것을 배웠다.

## AABB vs OBB: 회전 시 어떤 차이?
- AABB는 사각형이 회전할 때 벌어지는 모서리 부분까지 계산하여 회전하지 않는 AABB는 커지고 작아지고를 반복하는 것을 확인할 수 있었다. OBB는 사각형이 회전할 때 모서리 부분에 맞춰 같이 돌아서 빈 공간이 없는 것을 확인할 수 있었다.

## OBB가 회전하는 사각형 모서리에 맞게 돌지 않는 현상
- 코드를 실행 시키니 회전하는 사각형에 OBB가 알맞게 돌지 않아서 따로 노는 현상이 발생하였다. 이 문제를 해결하기 위해 Ai에게 이유를 물어보니 "fixed_surf를 회전시키고 화면에 그리는 방식과, OBB 꼭짓점 계산 방식이 동일 좌표계와 기준점으로 맞춰지지 않았기 때문" 이라는 답변을 해주어서 문제가 어느 부분이였는지 알 수 있게 되었다.
